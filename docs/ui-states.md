# UI State Matrix

## 说明

- 适用范围：首页、房源详情、发布房源、我的、登录注册、聊天等核心页面。
- 本文只定义用户可见反馈，不涉及任何实现方式。
- `empty` 状态统一理解为“组件承载的内容为空时，用户看到的完整占位反馈”，因此即便是输入框、按钮这类组件，也需要定义空内容呈现。
- 全局规则：
  - `loading` 使用骨架屏或 shimmer，不使用转圈图标，不显示“加载中”文字。
  - `error` 优先使用组件内或页面内联反馈；`toast` 只用于操作结果反馈。
  - `empty` 必须包含：插画/图标描述、标题、副标题，必要时提供 CTA。
  - `disabled` 统一为 `opacity 0.4`、不可点击、无悬浮提示。
  - `uploading` 仅在有上传语义的组件中启用；需要在图片上显示进度条和百分比覆盖层。

## 1. Page-level loading（初始数据加载）

| State | Visual | Copy | Behavior |
|---|---|---|---|
| default | 页面顶部标题、筛选区、内容区全部按正式布局展示；列表卡片、模块标题、按钮均为真实内容。 | 无固定文案。 | 页面可正常滚动、点击、下拉刷新。 |
| loading | 首屏区域显示整页骨架：顶部搜索条为长条 shimmer，筛选栏为圆角块骨架，列表为 3 到 5 张房源卡片骨架；详情页则显示大图、标题、价格、段落骨架。 | 无文案。 | 页面进入后立即展示骨架，占位尺寸与正式内容一致；骨架结束后平滑淡出进入真实内容。 |
| empty | 内容区中央显示插画：灰蓝色房屋轮廓配放大镜；标题“暂无房源”；副标题“换个条件试试”；主按钮“重置筛选”。 | “暂无房源”“换个条件试试”“重置筛选” | 页面其它结构保留，用户可直接点 CTA 重置条件或返回上一步修改搜索。 |
| error | 页面内容区显示内联错误卡片：浅红底、红色感叹号图标、说明文字和主按钮；背景布局仍保留，不跳空白页。 | 标题“加载失败”；副标题“网络开小差了”；按钮“重新加载” | 点击“重新加载”后错误卡片原位切回骨架，不弹系统级报错。 |
| disabled | 页面整体保持当前内容，但蒙上一层极浅遮罩，所有可操作元素透明度降为 0.4。 | 可选提示文案“当前暂不可操作”。 | 用户无法点击、输入或滚动到新交互；用于权限校验或关键流程锁定时短暂展示。 |
| active/selected | 当前页面或模块进入焦点态，例如首页搜索栏吸顶、当前 Tab 高亮、当前详情模块标题加深。 | 无新增文案。 | 强调当前浏览位置或当前被选中的分段，方便用户感知所在页面层级。 |
| uploading | 若页面承载上传任务，原页面内容保持可见，顶部不出现全局加载层；上传模块内部显示图片进度覆盖层。 | 页面级无新增文案。 | 用户仍可浏览其它已加载内容，只在具体上传区域看到上传进度变化。 |
| success | 骨架淡出后内容整体渐显；若是刷新成功，可在页面顶部短暂出现淡绿色细条反馈。 | 短提示“已更新” | 成功提示停留后自动消失，页面停留在最新内容状态，不打断继续浏览。 |

## 2. List（搜索结果、消息列表）

| State | Visual | Copy | Behavior |
|---|---|---|---|
| default | 房源列表显示缩略图、标题、租金、户型、区域标签；消息列表显示头像、昵称、最后一条消息、时间和未读点。列表项之间有稳定留白与分隔。 | 无固定文案。 | 支持点击进入详情、上拉加载更多、下拉刷新。 |
| loading | 列表区域用卡片式 shimmer 占位：图片块、两行标题块、价格块、标签块依次闪动；消息列表用头像圆形骨架和两行文本骨架。 | 无文案。 | 保持列表高度预期，避免页面跳动；加载完成后骨架原位替换为真实项。 |
| empty | 列表中部显示插画：空文件夹配小房子或对话气泡；标题“没有结果”；副标题“缩短关键词再试”；可选按钮“清空筛选”。 | “没有结果”“缩短关键词再试”“清空筛选” | 用户可点击 CTA 立即恢复默认列表；若无 CTA，则保留滚动但无列表项。 |
| error | 列表顶部或中部出现内联错误条，下面原列表若有缓存则继续保留；无缓存时显示错误占位卡片。 | 标题“列表出错”；副标题“请稍后再试”；按钮“重新加载” | 点击“重新加载”后错误卡片切回列表骨架；若有旧数据，用户仍可点旧数据进入详情。 |
| disabled | 列表整体降到 0.4 透明度，单条卡片点击态、下拉刷新、上拉加载全部失效。 | 无新增文案。 | 常用于筛选切换中或权限受限场景，列表内容只读不可交互。 |
| active/selected | 被按下的列表项出现轻微底色加深和缩略阴影；消息列表当前会话项带主色左边条或浅色底。 | 无新增文案。 | 松手后进入详情或会话；当前已进入的会话在返回列表时保持选中高亮。 |
| uploading | 不适用列表整体上传；若列表项内存在媒体发送，则仅该条项目显示图片进度覆盖层。 | 无固定文案。 | 其它列表项不受影响，用户仍可继续滚动和查看历史内容。 |
| success | 新数据刷新后，新增项短暂高亮背景 1 次；消息列表中新消息会轻微闪入。 | 短提示“已为你更新” | 列表停留在最新位置或按策略回到顶部；用户可立即继续操作。 |

## 3. Image（卡片缩略图、详情图集）

| State | Visual | Copy | Behavior |
|---|---|---|---|
| default | 图片按比例裁切，圆角完整，默认显示真实照片；图集当前图清晰可见，底部页码点正常。 | 无文案。 | 点击缩略图进入详情，点击详情大图可预览切换。 |
| loading | 图片区域显示与原尺寸一致的灰色 shimmer 块；图集首图与缩略图都先用骨架占位。 | 无文案。 | 用户可先浏览其它文字内容，图片加载完成后渐显，不发生布局抖动。 |
| empty | 图片区域显示插画：线框房屋加照片占位符；标题“暂无图片”；副标题“房东稍后补充”；无按钮。 | “暂无图片”“房东稍后补充” | 图片区域不可预览，页面其它模块继续可用。 |
| error | 图片显示浅灰底和断图图标，中央有重试型胶囊按钮。 | 标题“图片出错”；副标题“点一下重新载入”；按钮“重试” | 点击“重试”后原位回到图片骨架；多图场景只重试失败的那一张。 |
| disabled | 图片透明度降为 0.4，并覆盖半透明遮罩；图库页码点也同步变淡。 | 可选短文案“不可预览” | 不响应点击放大、不响应左右切换。 |
| active/selected | 当前缩略图外侧出现主色描边；图集当前页码点加深放大，选中缩略图可带 2px 高亮边框。 | 无新增文案。 | 用户滑动或点击切换时，选中态实时跟随更新。 |
| uploading | 图片上方覆盖半透明深色层，底部或中央显示进度条和百分比，例如 `68%`；进度条直接覆盖在当前图片上。 | “上传中 68%” | 上传未完成前可继续查看已上传图片，但当前图片不可再次选择；失败后切到错误态。 |
| success | 上传完成或加载成功后，进度层消失，右上角短暂出现绿色对勾徽标。 | 短提示“上传完成” | 成功徽标短暂停留后消失，图片恢复为普通可预览状态。 |

## 4. Button（Primary、Secondary、Ghost）

| State | Visual | Copy | Behavior |
|---|---|---|---|
| default | `Primary` 为实底高对比按钮；`Secondary` 为浅底描边按钮；`Ghost` 为透明底文字按钮。尺寸、圆角、内边距统一。 | 示例：“立即发布”“保存草稿”“稍后再说” | 点击后触发对应操作，并提供轻微按压反馈。 |
| loading | 按钮保持原尺寸，不出现转圈；内部文案位置替换为一条短 shimmer 条，按钮外形不变。 | 无固定文案。 | 当前按钮不可重复点击，页面其它无关操作可继续。 |
| empty | 当按钮承载的是“选择”或“进入下一步”但当前无可用目标时，按钮区域显示空操作占位样式：浅灰底、图标占位和空态说明。 | 标题“暂无操作”；副标题“当前没有可点项”；可选按钮“刷新” | 用户看得到按钮位置，但无法误触；如有 CTA，仅提供刷新可选项。 |
| error | 按钮恢复可点击，同时在按钮下方或右侧显示红色错误辅助文案；`Primary` 边框可短暂变红。 | “操作失败，请重试” | 用户可再次点击；如失败由校验引起，则焦点跳到相关字段。 |
| disabled | 三种按钮统一降到 0.4 透明度；`Primary` 仍保留填充色但明显发灰，`Secondary` 与 `Ghost` 都不可触发。 | 保留原文案，例如“下一步” | 不响应点击，无遮挡提示，无额外交互。 |
| active/selected | 按下瞬间按钮轻微缩小或底色加深；被选中的 `Ghost`/`Secondary` 可变为主色文字或主色描边。 | 保留原文案，例如“已选择” | 释放后执行操作；若是切换型按钮，保留选中态直到用户改选。 |
| uploading | 仅用于“上传图片”“提交认证照”等按钮时，按钮内显示细进度条或百分比，不弹全局遮罩。 | “上传中 42%” | 按钮不可重复点击，上传完成后切到 success。 |
| success | `Primary` 按钮短暂变为成功色，`Secondary`/`Ghost` 显示成功描边或成功文字色。 | “已保存”“已发送”“已完成” | 成功反馈停留后，按钮恢复默认或进入下一步可操作状态。 |

## 5. Input field

| State | Visual | Copy | Behavior |
|---|---|---|---|
| default | 输入框显示常规边框和占位符，左侧可带图标，右侧在有值时显示清空入口。 | 示例占位：“请输入小区/地铁/区域” | 点击后聚焦并调起输入；支持删除、粘贴、清空。 |
| loading | 输入框位置显示与实际宽高一致的 shimmer 条，占位符和边框都暂不出现。 | 无文案。 | 用户暂时不能输入，等待字段配置或默认值加载完成。 |
| empty | 字段为空时，输入框保留占位符；若该字段是必填且用户已尝试提交，则输入框下方出现空态提示卡。 | 插画：小表单图标；标题“未填内容”；副标题“输入后才能继续” | 用户点击字段后空态提示收起，转为正常编辑状态。 |
| error | 输入框边框和标题文字变红，底部显示单行错误说明。 | “请输入正确手机号”“验证码为 6 位数字” | 聚焦后错误态保留，直到内容修正；再次校验通过后恢复默认。 |
| disabled | 输入框透明度降为 0.4，背景变浅，光标不出现。 | 保留已填值或占位，例如“暂不可修改” | 不可点击、不可输入、不可清空。 |
| active/selected | 输入框边框加深或变主色，光标明显；有内容时清空按钮显示。 | 保留用户正在输入的内容。 | 键入时实时显示字符；失焦后按校验结果切换到 default 或 error。 |
| uploading | 仅在输入框承载上传型内容时使用，例如图片链接自动回填；字段右侧显示进度条小标签。 | “上传中 75%” | 用户可继续看表单，但不能手改该字段内容，直到上传完成。 |
| success | 输入框边框变为成功色，右侧显示小对勾。 | “填写正确” | 成功态短暂停留，随后回到默认可编辑状态。 |

## 6. Form（多字段表单）

| State | Visual | Copy | Behavior |
|---|---|---|---|
| default | 多个字段按逻辑分组展示，组标题清晰，字段间距统一；底部有主按钮和次按钮。 | 组标题示例：“基础信息”“房源描述” | 用户可顺序填写、跳项填写、滚动查看全部字段。 |
| loading | 表单整体显示分组骨架：标题短条、输入框长条、上传框方块、底部按钮骨架。 | 无文案。 | 在默认值或编辑数据拉取完成前，用户不能输入，避免看到半成品字段。 |
| empty | 表单区域显示插画：剪贴板和虚线表单框；标题“暂无内容”；副标题“先补充基础信息”；主按钮“开始填写”。 | “暂无内容”“先补充基础信息”“开始填写” | 点击 CTA 后进入第一个可编辑字段并自动聚焦。 |
| error | 表单顶部出现错误摘要条，点名缺失项数量；各错误字段原位变红。 | 顶部文案“还有 3 项待完善”；字段文案如“请填写月租金” | 点击顶部摘要可滚动到首个错误字段；提交按钮保持可见。 |
| disabled | 整个表单降至 0.4 透明度，字段、上传区、底部按钮全部不可点。 | 可选说明“审核中暂不可编辑” | 表单内容只读展示，用户无法改动任何项。 |
| active/selected | 当前编辑字段所在分组标题高亮，当前卡片块阴影略增强。 | 无新增文案。 | 帮助用户在长表单中定位当前填写位置。 |
| uploading | 表单中只有上传模块进入上传态，其余字段保持可见；顶部不出现全局遮罩。 | 无新增文案。 | 用户可继续填写文本字段，上传完成后再提交整个表单。 |
| success | 提交通过后表单顶部显示成功横幅，底部主按钮切成成功样式。 | “信息已保存” | 成功后可进入下一步、留在当前页继续编辑，或自动跳转到成功页。 |

## 7. Upload area（图片选择区）

| State | Visual | Copy | Behavior |
|---|---|---|---|
| default | 显示虚线边框的图片选择框，内有相机或图片图标；已选图片以网格卡片展示，末尾保留“添加”入口。 | “上传房源图片”“最多 9 张” | 点击添加可选图，点击已有图片可预览、删除、重排。 |
| loading | 上传区本身显示方形 shimmer 占位格，模拟未来图片数量和排布。 | 无文案。 | 在历史图片或草稿数据载入前，用户能预期图片区占位大小。 |
| empty | 上传区中央显示插画：相册轮廓和加号；标题“暂无图片”；副标题“至少上传一张”；主按钮“选择图片”。 | “暂无图片”“至少上传一张”“选择图片” | 点击 CTA 进入图片选择；若用户跳过，提交时转为 error。 |
| error | 失败图片卡片保留原位置，但覆盖红色半透明层和错误标签；底部出现简短错误说明。 | “上传失败”“点按重新上传” | 用户可单独重试失败图片，也可删除失败图片重新选图。 |
| disabled | 整个上传区透明度降为 0.4，添加按钮和删除按钮都不可用。 | 保留原文案，如“上传房源图片” | 用户仅可查看现有图片，不可新增或删除。 |
| active/selected | 被选中的图片卡片出现主色描边和勾选角标；“添加”入口按下时底色加深。 | 无新增文案。 | 可执行预览、排序、删除等针对性操作。 |
| uploading | 每张上传中的图片都覆盖进度条和百分比，例如 `23%`、`81%`；覆盖层直接叠在该图片上。 | “上传中 23%” | 仅对应图片进入上传态，其它图片仍可预览；上传完成后单张转 success。 |
| success | 新上传完成的图片卡片右上角短暂出现成功对勾，图片清晰显示；空上传位自动后移。 | “上传完成” | 成功图片立即并入正式图片列表，可继续追加下一张。 |

## 8. Step indicator（发布向导）

| State | Visual | Copy | Behavior |
|---|---|---|---|
| default | 三步横向排列：数字圆点 + 步骤名；未开始步骤为浅灰线与浅灰字。 | “1 基础信息”“2 房源图片”“3 租金入住” | 用户能理解总共三步，以及当前尚未进行到哪一步。 |
| loading | 步骤条位置显示三段式 shimmer，占位当前与未来步骤。 | 无文案。 | 用于页面首次进入时，避免步骤条跳动。 |
| empty | 向导尚未开始时，步骤条下方显示小空态卡片。 | 插画：路径旗帜图标；标题“尚未开始”；副标题“先填写第一步”；按钮“开始发布” | 点击 CTA 后进入第一步，第一节点转 active。 |
| error | 当前步骤圆点变红，步骤名下方出现一行错误摘要。 | “本步未完成”“请补全必填项” | 用户点击“下一步”失败后留在当前步，错误字段同步高亮。 |
| disabled | 未解锁的后续步骤透明度降为 0.4，节点与连线都发灰。 | 保留原步骤名。 | 用户无法直接跳到未完成前置条件的步骤。 |
| active/selected | 当前步骤主色高亮，圆点放大，连线填充到当前位置。 | 当前步骤名加粗，如“2 房源图片” | 用户始终知道正在进行哪一步；点击已完成步骤可回看修改。 |
| uploading | 当第二步上传图片时，第二节点下方出现细进度提示条。 | “图片上传中 60%” | 当前步骤保持可见，用户仍知道上传发生在第几步。 |
| success | 已完成步骤变为成功色并显示对勾，连线同步点亮；最后一步完成后整条步骤线全部激活。 | “已完成” | 用户可继续进入下一步；全部完成后转入提交成功页。 |

## 10. Bottom action bar

| State | Visual | Copy | Behavior |
|---|---|---|---|
| default | 底部固定操作栏包含主操作和次操作，如“联系房东”“收藏”“下一步”“提交发布”；安全区完整留白。 | 示例：“联系房东”“收藏”“下一步”“提交发布” | 用户滚动页面时操作栏固定可见，方便随时执行关键动作。 |
| loading | 操作栏位置显示整条骨架，按钮区域为左右两个或三个圆角 shimmer 块。 | 无文案。 | 用于详情页或表单页首屏数据未到时，先稳定底部布局。 |
| empty | 当当前内容没有可执行操作时，操作栏显示空态占位。 | 插画：手势按钮图标；标题“暂无操作”；副标题“当前内容不可操作”；可选按钮“返回首页” | 用户不会误以为界面异常；如有 CTA，可引导离开当前无效场景。 |
| error | 某个操作失败后，操作栏上方出现一条内联错误条，主按钮恢复可点。 | “操作未完成”“请稍后重试” | 用户可立即再次点击；若失败由表单问题引起，则页面自动滚到对应位置。 |
| disabled | 整个底栏透明度降为 0.4，所有按钮失效。 | 保留原按钮文案，如“提交发布” | 不响应点击，不显示额外提示。 |

| success | 主按钮短暂切为成功色，次操作恢复常态；必要时底栏上方出现轻提示条。 | “发布成功”“已发送” | 成功后进入成功页、详情页或保留当前页继续下一步动作。 |
