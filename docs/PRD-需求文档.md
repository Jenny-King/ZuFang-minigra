# 租房微信小程序 PRD

## 1. 项目概述

### 1.1 项目名称

- 租房微信小程序

### 1.2 项目目标

- 为租客提供便捷、稳定的找房与咨询体验
- 为房东提供低成本的房源发布与管理能力
- 建立清晰、可维护、可扩展的小程序项目结构
- 建立稳定的登录、绑定、会话鉴权体系

### 1.3 产品定位

- 产品形态：微信小程序
- 核心方向：租房信息展示、收藏、咨询、发布管理
- 当前目标：优先完成核心业务闭环，不引入非核心复杂功能

## 2. 用户角色

### 2.1 租客

- 浏览房源列表
- 查看房源详情
- 收藏房源
- 查看浏览历史
- 与房东聊天咨询

### 2.2 房东

- 发布房源
- 编辑房源
- 删除房源
- 接收租客咨询消息

### 2.3 管理员

- 保留基础管理入口
- 不作为第一阶段重点交互角色

## 3. 业务目标

### 3.1 核心业务闭环

1. 用户登录 / 注册
2. 浏览房源
3. 查看房源详情
4. 收藏或咨询
5. 房东发布和维护房源

### 3.2 第一阶段交付目标

- 首页找房稳定可用
- 房源详情完整可读
- 发布与编辑链路闭环
- 登录注册资料链路统一
- 聊天基础能力可用
- 收藏与历史稳定可用
- 微信绑定 / 解绑逻辑清晰可控

## 4. 功能范围

### 4.1 首页找房

功能说明：

- 展示房源列表
- 支持关键词搜索
- 支持区域、价格、户型、排序筛选
- 支持分页加载

验收标准：

- 用户进入首页后可看到房源列表
- 搜索和筛选后结果正确刷新
- 列表空状态和加载状态明确

### 4.2 房源详情

功能说明：

- 展示房源图片、租金、户型、面积、地址、描述、配套
- 展示地图位置和周边设施
- 支持收藏 / 取消收藏
- 支持记录浏览历史
- 支持联系房东进入聊天

验收标准：

- 用户能完整看到房源详情信息
- 收藏和历史记录写入正确
- 点击联系房东可进入聊天页

### 4.3 房源发布与管理

功能说明：

- 房东可新增房源
- 房东可编辑已有房源
- 房东可删除房源
- 表单包含基础信息、配套、定位、联系方式

验收标准：

- 发布后首页和详情页可正常展示
- 编辑后数据立即更新
- 删除后房源不再出现在列表中

### 4.4 登录注册

功能说明：

- 支持微信授权登录
- 支持手机号验证码登录
- 支持手机号密码登录
- 支持注册时完善个人资料
- 登录成功后返回有效 session

验收标准：

- 登录成功后可恢复用户状态
- 新用户首次进入可完成注册并进入业务页
- 失败场景有明确错误提示
- 不会再出现“两个账号共享同一个 `_openid` 导致串号”

### 4.5 用户资料与账号

功能说明：

- 查看当前账号资料
- 修改昵称、微信号、地区、密码
- 显式绑定微信
- 显式解绑微信
- 切换角色
- 退出登录
- 注销账号

当前阶段不纳入：

- 手机号直接修改 / 换绑流程

验收标准：

- 资料修改后立即生效
- 密码修改成功后可重新登录
- 微信绑定冲突时有明确提示
- 退出登录后本地会话清除
- 注销后服务端身份和会话同步失效

### 4.6 实名认证

功能说明：

- 提交真实姓名与身份证号
- 更新认证状态

验收标准：

- 合法信息可提交成功
- 非法身份证格式拦截
- 前端仅展示脱敏结果

### 4.7 收藏与浏览历史

功能说明：

- 查看收藏列表
- 取消收藏
- 查看浏览历史
- 删除单条历史
- 清空历史

验收标准：

- 收藏和历史列表与当前用户正确关联
- 删除和清空操作反馈明确

### 4.8 聊天与通知

功能说明：

- 用户与房东一对一聊天
- 显示会话列表
- 显示未读数
- 展示系统通知和业务通知

当前阶段刷新策略：

- 先采用定时轮询
- 聊天详情页进入后按固定间隔轮询消息
- 页面离开后立即销毁轮询
- 会话列表页在 `onShow` 时刷新未读状态

验收标准：

- 可正常进入会话详情收发消息
- 会话列表可显示最后一条消息和未读数
- 通知按时间倒序展示

## 5. 非功能需求

### 5.1 可维护性

- 页面、服务层、工具层、状态层职责分离
- 云函数按业务域拆分
- 所有业务调用统一经由 `services` 层

### 5.2 稳定性

- 登录态恢复稳定
- token 失效时能清理本地会话
- 初始化逻辑不能重复灌测试数据

### 5.3 安全性

- 密码必须加密存储
- 身份证信息不得明文返回前端
- 敏感操作必须校验当前登录用户权限
- 服务端当前用户识别统一走 session

### 5.4 性能

- 主包只保留核心高频页面
- 低频页面使用分包
- 列表页和聊天页避免重复请求

## 6. 技术约束

- 微信小程序原生开发，不使用 `uni-app`、`Taro`
- 使用微信云开发：云函数、云数据库、云存储
- 所有异步逻辑统一使用 `async/await`
- 云函数调用统一封装，页面禁止直接调用 `wx.cloud.callFunction`
- 页面层只负责 UI 和交互，不直接处理复杂业务逻辑
- 服务层统一处理参数、错误、权限和返回值适配
- 不允许在 `app.js` 启动阶段自动插入测试数据
- 统一用户身份模型：
  - `userId` 作为业务主键
  - `user_identities` 负责手机号 / 微信身份绑定
  - `user_sessions` 负责登录会话
  - 客户端保存 `accessToken + userInfo`

## 7. 页面范围

### 7.1 核心页面

- 首页：房源列表
- 房源详情页
- 发布 / 编辑页
- 我的房源页
- 聊天页
- 聊天详情页
- 我的页
- 登录页
- 注册页
- 实名认证页
- 收藏页
- 历史页

### 7.2 暂不纳入本期

- 手机号换绑
- 预约看房
- 示例页面
- 测试数据生成入口
- 非核心后台管理能力

## 8. 成功标准

- 用户可以完成登录、找房、查看详情、收藏、聊天咨询完整闭环
- 房东可以完成发布、编辑、删除房源闭环
- 微信身份绑定不再与手机号账号混淆
- 重构后代码结构清晰，页面不再直接依赖云函数细节
- 关键敏感数据处理符合基本安全要求
